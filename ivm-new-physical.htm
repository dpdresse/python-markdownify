Instant Volume Mount to a New Location for Physical Server

25 June 2024

To perform instant volume mounting to the original server:

  1. Select Data Protection > Recoveries.

  2. Click on Recover located at the top right of the page and then select Physical Server> Instant Volume Mount.

    The Instant Volume Mount page appears.

  3. Search for the server that contains the data you want to mount by entering characters of the server name or by specifying the wildcard character *.

    A list of servers displays.

    You can optionally filter the search results by source, Protection Group, Storage Domain, server type, or date.

  4. From the list of search results, select the server that contains the data you want to mount.

    By default, the latest snapshot is selected for recovery.

  5. To recover from a different snapshot, mouse over the snapshot from the selected list, and then click the edit icon.

    The Edit recovery point for <server_name> page appears.

    In this workflow, deleted snapshots may be displayed as valid recover points for a brief period until they are removed from the search index by a background process.

    By default, the option to select the snapshot is displayed in the Timeline mode. You can also choose to view the option in List mode by clicking the List tab in the top-right corner of the page.

    • To select a snapshot from the Timeline mode:

      1. From the Choose a date drop-down, select a date.

        The snapshots available on the selected date are displayed in blue dots on the timeline bar.

      2. Select the snapshot to be recovered by clicking the corresponding blue dot.

      3. By default, the snapshot is recovered from the local Cohesity cluster. You can also select the snapshot archived to the cloud by clicking the cloud () icon if available in the Location option.

      4. Click Select Recovery Point and then go to step 6.

    • To select a snapshot from the List mode:

      1. Select the List tab from the top-right corner of the Edit recovery point for <server_name> page.

        By default, the page displays the snapshots available in the last one month.

      2. You can also select a snapshot from a different date range by selecting the date range from the Date drop-down list. The page displays the available snapshots in the specified range.

      3. Select the snapshots to be recovered.

      4. By default, the snapshot is recovered from the local Cohesity cluster. You can also select snapshots archived to the cloud by clicking the cloud () icon if available in the Location option.

      5. Click Select Recovery Point and then go to step 6.

  6. Click Next: Recover Options.

  7. From the Select Volume(s) option, perform one of the following actions:

    • Enable Select all Volumes to recover all the volumes of the selected server

    • From the Select one or more options drop-down list, select the individual volumes you want to recover.

    After volumes are mounted, they may be displayed as different volumes than their source volume. For example, D: volume may be mounted as X: volume.

  8. From the Recover To options, select New Location to recover the volume to the original location.

  9. From the Source drop-down menu, select a registered source having the server to which you want to recover the volume. You can also register a new source by clicking Register Source from the Registered Source drop-down menu.

  10. From the Target drop-down menu, select a physical server to which the volume will be mounted.

    To mount to a physical server, the server must already have the Cohesity agent installed and registered with the Cohesity cluster. If not, the server will not be available as a mount target.

  11. (Applies only for VMs) With the Ensure Disks are Online option enabled, select one of the following options to deploy Cohesity Agent on the target VM.

    For Linux target VMs, you cannot disable this option. For Window target VMs, you can disable this option.

    • Auto Deploy Cohesity Agent: Select this option and provide the username and password of the target VM to automatically deploy the Cohesity Agent on the target VM.

    • Use Existing Cohesity Agent: Select this option to use the existing Cohesity Agent deployed on the target VM

      The following table provides information on how the volume is mounted when the Ensure Disks are Online option is enabled and disabled.

      Environment Ensure Disks are Online option is Turned On Ensure Disks are Online option is Turned Off
      VMware

      After the VMDKs are attached to the VM, Cohesity cluster explicitly mounts the volume. To mount the volume to the VM, Cohesity cluster requires the built-in administrator credentials, such as the local system administrator of the VM. It uses the credentials provided in the Mount Target Credentials fields for mounting the volume.

      If the UAC property "Admin Approval mode" is disabled, then you can specify any user who is part of the system’s Local Administrator group.

      • VMware Tools are required to bring the disks online, but mounting is still supported without VMware Tools.
      		 The VMDKs are just attached to the VM and the VM may bring the disks online. Cohesity cluster does not explicitly bring the volumes online within the VM. In this case, the Target Volume displays as "n/a" after mounting the volume succeeds.
      Standalone Hyper-V host or cluster

      You must add the guest VM to the reverse lookup zone in the DNS server to complete the recovery successfully.

      		 Not Applicable

  12. Optional. Edit the default settings of the following Recovery Options:

    • Cluster Interface: By default, the Auto Select option is enabled and the recovery task automatically selects the correct VLAN. If you disable this option, then select a configured interface group from the Interface Group drop-down.

    • Task Name: Change the default name of the recover task.

  13. Click Recover.

The recovery task is initiated. You can monitor the recovery task from the Recoveries page.

If a drive letter was provided as the file path for a file-based backup of NTFS volumes, the default attributes for drive letters were preserved, so hidden drives remain hidden. They are still recovered and can be accessed by providing the full path in search bar.
